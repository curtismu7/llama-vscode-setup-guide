🤔 What is this thing?

Imagine a robot helper that lives inside your code editor and can read your files, write code, and run commands for you — like a super-smart autocomplete that can also do whole tasks.

llama-vscode is a free add-on for VS Code (a popular code editor). It connects to an AI "brain" that runs on your own computer, so it's private and free to use.

Think of it like a car 🚗 The extension is the dashboard and steering wheel. The model (brain) is the engine. You need both — and you have to turn the engine on before you can drive.
1. Install dashboard 2. Install engine starter 3. Get an engine 4–6. Plug it in & start it 7. Drive! 🏁

🧰 What you need first

  • ✅ A Mac (this guide uses Mac keys). Ideally Apple Silicon (M1/M2/M3/M4) with 16 GB+ memory.
  • ✅ VS Code installed — get it free at code.visualstudio.com
  • ✅ Homebrew (a tool installer for Mac) — get it at brew.sh
  • ✅ About 10–15 GB free disk space (AI brains are big files)

1Install the extension (the dashboard)

Open VS Code. On the left side, click the Extensions icon (it looks like 4 squares 🔲). Search for llama-vscode and click Install.

2Install the model runner (the engine starter)

The extension needs a little program called llama.cpp to actually run the AI brain. Open the Terminal app (or VS Code's terminal: menu Terminal → New Terminal) and paste this, then press Return:

brew install llama.cpp

When it finishes, check it worked:

llama --version

If you see a version number (not "command not found"), you're good. ✅

What did that do? It installed the llama command — the thing that loads an AI brain file and lets the extension talk to it.

3Get an AI model (the brain) 🧠

AI "brains" are files called models (they end in .gguf). You have two easy ways to get one:

Way A — Let llama download it for you (easiest)

You don't have to do anything now! In Step 6 you'll pick a ready-made setup, and llama will automatically download the brain the first time. Just be patient — the first download is several gigabytes.

Way B — Download it yourself into a folder

Make a models folder in your home directory and download a brain into it. Example (a small, fast one — about 2 GB):

mkdir -p ~/models
hf download unsloth/gemma-3-4b-it-qat-GGUF gemma-3-4b-it-qat-Q4_0.gguf --local-dir ~/models
Which brain should I pick?
  • For chatting / asking questions: a Gemma or Qwen model.
  • For "vibe coding" (the robot that edits files): gpt-oss 20B is the best free local choice. ⭐
  • Bigger number (20B, 14B) = smarter but slower & needs more memory. Smaller (4B, 1.5B) = faster but simpler.
⚠️ Careful where you "download" from a web page If you click a download button on a website and end up with a tiny file (a few hundred KB) instead of gigabytes, you accidentally saved the web page, not the brain. Use the hf download command above instead — it always gets the real file.

4Open the llama menu

This is where most people get stuck, so read carefully. There are two ways to open llama-vscode's controls:

The easy way: the Command Palette

Press Cmd+Shift+P. A search box appears at the top. Type llama. You'll see a list of llama commands — click the one you want.

Command Palette — type "llama"
›llama
⚙️llama-vscode: Show Menu (pick your models)⌃⇧M
🤖llama-vscode: Show Llama Agent (the vibe-coding robot)⌃⇧A

The other way: the status bar

Look at the very bottom strip of VS Code. There's a little 🦙 llama button. Click it to open the menu.

🦙 llamamainUTF-8click the llama 👆
⚠️ The #1 gotcha: keyboard shortcuts use Control, not Command! The symbol ⌃ means the Ctrl key — not ⌘ Cmd. So it's:
  • Ctrl+Shift+M = open the Menu
  • Ctrl+Shift+A = open the Agent
If you press Cmd+Shift+M by mistake you'll get the wrong thing (a terminal/panel). When in doubt, use Cmd+Shift+P → type "llama" instead — that always works.

6Turn the engines on 🔌

Each brain runs as a tiny background "server" on a numbered port (like a parking spot number). When you pick a model, llama-vscode offers to start its server — say yes. The first time, it may download the brain (be patient).

You can also start one yourself in the Terminal. Example:

llama serve -m ~/models/gemma-3-4b-it-qat-Q4_0.gguf --port 8011
JobUsual port (parking spot)
Tools / agent brain (gpt-oss)8009
Embeddings8010
Chat (small brains)8011
Chat (big brains)8012
⚠️ "couldn't bind … port 8012" — what that means It means a brain is already running in that parking spot. You don't need to start it twice! Either just use the one that's running, or stop it first with: 
lsof -ti:8012 | xargs kill
(Swap 8012 for whatever port the error mentions.)

7Do "vibe coding"! 🏁

Open the robot: Ctrl+Shift+A (or Command Palette → "llama-vscode: Show Llama Agent"). A panel opens on the side with a text box.

🤖 Llama Agent
Type what you want the robot to do…@ to attach a file
🧑 You: "Read server.js and tell me what it does in 5 bullets."
🤖 Robot: reading file… ✓ → writes the answer

Your first prompts (start safe → then edit)

  1. Safe test (just reads): "Read server.js and give me a 5-bullet summary of what it does."
  2. Then a small edit: "Add a /health route to server.js that returns status ok. Show me the change before saving."
Golden rules for happy robots
  • 🎯 Be specific. "Add X to file Y" works way better than "make my code better."
  • 📎 Use the @ button to point it at the exact file — saves it from hunting around.
  • 💾 Save your work in Git first so you can undo anything the robot changes.
  • 👀 Read before you approve when it wants to run a terminal command.

🚑 When things break

Shortcut does the wrong thing

You probably pressed Cmd instead of Ctrl. The llama shortcuts use Ctrl. Or just use Cmd+Shift+P → type "llama".

"couldn't bind … port"

That brain is already running. Use it as-is, or free the spot: lsof -ti:8012 | xargs kill (use the port from the message).

A download got stuck / keeps failing ("416")

A broken half-download is jammed. Delete it and try again:

find ~/.cache/huggingface -name '*.downloadInProgress' -delete

Then re-run your hf download … command.

My model file is tiny (KB instead of GB)

You saved a web page by accident. Delete it and use hf download … to get the real .gguf.

The robot prints weird symbols like <|channel|>

That's the gpt-oss brain leaking its internal "thinking" tags — harmless. It usually means the chat template settings need a tweak; it doesn't affect the actual edits. You can ignore it, or ask a techy friend to adjust the model's launch flags.
