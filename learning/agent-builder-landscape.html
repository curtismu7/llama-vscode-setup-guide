LangChain

LangChain is the most widely adopted open-source framework for building LLM-powered applications and agents. It provides composable primitives — chains, agents, tools, memory — that work with any LLM provider. Current stable is LangChain 1.x (GA October 2025); the Super Banking demo's langchain_agent/ pins the older LangChain 0.3.x to orchestrate the Banking Agent's tool calls via LCEL.

1. LCEL — LangChain Expression Language

  • Pipe syntax for composing chains: chain = prompt | llm | parser
  • Lazy evaluation — chains are DAGs, not eager function calls
  • Built-in streaming, batching, and async support
  • Replaces the older LLMChain / SequentialChain API
from langchain_core.prompts import ChatPromptTemplate from langchain_openai import ChatOpenAI prompt = ChatPromptTemplate.from_template("Answer: {question}") llm = ChatOpenAI(model="gpt-4o") chain = prompt | llm # LCEL pipe result = chain.invoke({"question": "What is RFC 8693?"})

2. Agents & Tool Use

  • Agents decide which tool to call based on the LLM's output
  • Tool: any Python function decorated with @tool or wrapped in Tool()
  • ReAct pattern: Reason → Act → Observe loop (default for create_react_agent)
  • This demo uses create_tool_calling_agent (the pattern for the 0.3.x version this demo pins); current LangChain 1.x recommends create_agent or LangGraph instead
from langchain.agents import create_react_agent, AgentExecutor from langchain_core.tools import tool @tool def get_balance(account_id: str) -> str: """Get the balance for a given account.""" return "£4,250.00" agent = create_react_agent(llm, [get_balance], prompt) executor = AgentExecutor(agent=agent, tools=[get_balance]) executor.invoke({"input": "What is my balance?"})

3. LangGraph — Stateful Agent Workflows

  • Graph-based state machine for multi-step, multi-agent workflows
  • Nodes = functions/agents; Edges = conditional transitions
  • Supports cycles (loops), persistence (checkpointers), human-in-the-loop
  • Key concepts: StateGraph, MessagesState, interrupt_before, MemorySaver
  • Supersedes older AgentExecutor for complex workflows

4. LangSmith — Observability & Evaluation

  • Tracing: every LLM call, tool call, and chain step logged with latency + tokens
  • Evaluation: run datasets through chains, score outputs, regression test
  • Playground: interactive prompt iteration in the UI
  • Requires LANGCHAIN_API_KEY and LANGCHAIN_TRACING_V2=true

5. LangServe — Deployment

  • Turns any LangChain chain/agent into a FastAPI endpoint with one decorator
  • Auto-generates /invoke, /stream, /batch, /stream_log routes
  • Being superseded by LangGraph Platform for production deployments
Version note: LangChain 0.3.x split into langchain-core (stable primitives),langchain (agent abstractions), langchain-community (integrations), and provider packages like langchain-openai, langchain-groq, langchain-google-genai. Import paths changed significantly from 0.1.x. This demo pins 0.3.x; current stable is LangChain 1.x (GA October 2025), which recommends create_agent/LangGraph over hand-built LCEL agent chains.

Open-Source

Open-source agent builder alternatives to LangChain — each with a distinct philosophy and primary use case.

LlamaIndexRAG & data-first agents

Turn your data into AI-queryable knowledge

  • Focus: connecting LLMs to data sources (PDFs, databases, APIs, web). Best-in-class for RAG pipelines.
  • Key primitives: VectorStoreIndex, QueryEngine, RouterQueryEngine, SubQuestionQueryEngine
  • Agents: FunctionCallingAgent, ReActAgent — tool use on top of indexed data
  • 2024+ evolution: llama-index-core split; LlamaCloud for managed indexes
  • Best for: document Q&A, knowledge bases, enterprise data agents
Microsoft AutoGenMulti-agent conversation

Orchestrate multiple AI agents in collaborative conversations

  • Focus: multi-agent systems where agents converse with each other to solve tasks
  • Key primitives: ConversableAgent, AssistantAgent, UserProxyAgent, GroupChat
  • Unique: agents can execute code (Python, shell) in sandboxed environments
  • AutoGen 0.4+: redesigned with AutoGen Core for event-driven agent networks
  • Microsoft has since merged AutoGen with Semantic Kernel into the unified Microsoft Agent Framework (preview October 2025, 1.0 April 2026) — new projects should target it, with AutoGen continuing as a research-oriented option
  • Best for: data analysis pipelines, code generation, research workflows with multiple collaborating agents
CrewAIRole-based multi-agent orchestration

Crews of AI agents with roles, goals, and tasks

  • Focus: structured teams — define Agents with roles (researcher, writer, analyst), assign Tasks, assemble into a Crew
  • Key primitives: Agent(role=, goal=, backstory=), Task(description=, agent=), Crew(agents=, tasks=, process=)
  • Processes: Sequential (linear), Hierarchical (manager delegates), Parallel
  • CrewAI Enterprise: hosted control plane with triggers and monitoring
  • Best for: content pipelines, research-to-report workflows, long-horizon tasks with clear role separation
HaystackNLP pipeline framework

Production-ready NLP pipelines since before LLMs existed

  • Focus: modular pipeline graphs with Components; strong NLP roots (QA, summarization, NER)
  • Key primitives: Pipeline, Component, Generator, Retriever, DocumentStore
  • Haystack 2.0 (2024): fully redesigned; async-first, typed dataclasses, YAML serializable pipelines
  • Integrations: Weaviate, Qdrant, OpenSearch, Elasticsearch, PgVector
  • Best for: production NLP/RAG pipelines; teams wanting declarative, serializable pipeline definitions
HuggingFace smolagentsLightweight code-executing agents

Minimal, transparent agents that run HuggingFace models

  • Focus: agents that write and execute Python code to solve tasks (Code Agent pattern)
  • Key primitives: CodeAgent, ToolCallingAgent, Tool, HfApiModel
  • Philosophy: agents write Python snippets as actions — more flexible but requires a sandboxed execution environment
  • Best for: research, learning agent internals, HuggingFace model-centric workflows
Phidata / AgnoFull-stack agent framework

Agents with memory, knowledge, tools, and reasoning baked in

  • Focus: production agents with built-in memory (PostgreSQL/SQLite), knowledge bases (PDFs, URLs, tables), structured outputs
  • Phidata rebranded to Agno in early 2025
  • Key primitives: Agent(model=, tools=, knowledge=, storage=), Team for multi-agent
  • Best for: full-stack agents that need persistent memory + knowledge base without assembling primitives manually

Commercial

Cloud-hosted and enterprise agent builder platforms — managed infrastructure with platform-specific integrations.

AWS Bedrock AgentsManaged cloud agent service

Build agents without managing infrastructure — AWS-native

  • Action Groups (Lambda-backed tool definitions), Knowledge Bases (S3 + OpenSearch RAG), Guardrails (content safety)
  • Agent collaboration: multi-agent orchestration via supervisor agents (2024)
  • Model choice: Claude, Llama, Titan, Mistral via Amazon Bedrock
  • Auth: IAM-native — cross-account agent invocation via resource-based policies
  • Best for: AWS shops; compliance-heavy orgs (FedRAMP, HIPAA, SOC 2)
Microsoft Copilot StudioLow-code agent builder

Build custom copilots — from no-code to pro-code

  • Generative AI topics (GPT-4o backbone), Plugin actions (OpenAPI connectors), Power Automate flows
  • MCP support added 2025; publish to Teams, web, phone; autonomous triggers
  • Enterprise features: DLP policies, Azure AD integration, audit logs, usage analytics
  • Semantic Kernel: the pro-code SDK underneath for custom plugins
  • Best for: Microsoft 365 shops extending Copilot with company-specific data and workflows
Google Vertex AI Agent BuilderManaged cloud agent service

Ground agents in your data with Google Search quality

  • Data Stores (website, structured, unstructured), Grounding (Google Search + own data), Playbooks (declarative orchestration)
  • Extensions (tool use via OpenAPI), Agent Engine (managed LangChain/LlamaIndex runtime)
  • Evolved from Dialogflow CX + CCAI in 2024
  • Best for: customer-facing agents needing Google Search grounding; GCP shops; Gemini long-context
Salesforce AgentforceEnterprise CRM-native agent platform

AI agents that take action in your CRM — no prompt engineering

  • Agent topics + instructions (declarative), Standard actions (CRM CRUD, flows, Apex), Atlas reasoning engine
  • Data Cloud grounding (360° customer data); built-in trust layer (PII masking, audit)
  • Agentforce 2.0 (late 2024): background agents, multi-agent coordination, MuleSoft API action library
  • Best for: sales, service, and marketing orgs already on Salesforce; CRM-centric agent workflows
DifyOpen-source + SaaS LLM app platform

LLM app platform with visual orchestration — open-source or cloud

  • Workflow canvas (drag-drop), Chatbot / agent modes, RAG pipeline (native chunking + retrieval)
  • Model provider hub (100+ models), API / iframe embed, self-hostable (Docker)
  • Popular in Asia-Pacific market; large open-source community
  • Best for: teams wanting a visual workflow builder; quick internal tool deployment; self-hosted LLM apps
n8nWorkflow automation + AI agents

Automate anything — visual workflows with AI Agent nodes and 400+ app integrations

  • Visual workflow builder with native AI Agent node (LangChain-backed); supports tool-calling via connected nodes
  • 400+ integrations (Slack, Salesforce, GitHub, databases) usable as agent tools out of the box
  • MCP support added 2025 — connect n8n workflows as MCP tools for any agent framework
  • Fair-code license (self-hostable free; n8n Cloud for SaaS); large OSS community
  • Best for: ops automation with embedded AI; teams who need broad integrations without custom tool code

How to choose

  • Learning / prototyping: LangChain — largest community, most tutorials, works with every LLM.
  • Production RAG pipelines: LlamaIndex or Haystack — designed around data ingestion, chunking, retrieval quality.
  • Multi-agent research workflows: AutoGen or CrewAI — AutoGen if you need code execution; CrewAI for structured role/task definitions.
  • Full-stack with persistence: Phidata / Agno — memory, knowledge base, structured outputs out of the box.
  • AWS cloud-native: Bedrock Agents — no infra, IAM auth, Guardrails, tight AWS integration.
  • Microsoft / Teams: Copilot Studio — publish to Teams, Power Automate integration, Azure AD governance.
  • Salesforce CRM: Agentforce — only platform with native CRM action library and Data Cloud grounding.
  • Visual / no-code: Dify — drag-and-drop workflow canvas, self-hostable, broad model support.
  • Ops automation + AI: n8n — 400+ integrations as agent tools, AI Agent node, MCP support, self-hostable.
  • This demo: langchain_agent/ uses LangChain 0.3.x with create_tool_calling_agent, tool functions calling BFF REST routes, and LangSmith tracing.
