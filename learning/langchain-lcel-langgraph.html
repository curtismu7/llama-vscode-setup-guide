Overview

What is LangChain?

LangChain is an open-source framework for building LLM applications. LCEL (LangChain Expression Language) — a declarative pipe-syntax for composing prompts, models, tools and output parsers into streaming-first chains — was introduced in 2023, before the first 0.1 release (Jan 2024), and carried forward through 0.2.x/0.3.x. This demo pins LangChain 0.3.x; the current stable release is LangChain 1.x (GA October 2025).

LCEL in one line

chain = prompt | llm.bind_tools(tools)

bind_tools attaches your MCP tools to the LLM. The model decides which tools to call; you run the tool-calling loop yourself or let LangGraph manage it.

Model-agnostic by design

Each provider ships its own langchain-* package. You swap the LLM object — the rest of the chain stays the same.

| Provider   | Package                  | Default model              |
|------------|--------------------------|----------------------------|
| Groq       | langchain-groq           | llama-3.1-8b-instant       |
| OpenAI     | langchain-openai         | gpt-4o-mini                |
| Anthropic  | langchain-anthropic      | claude-haiku-4-5-20251001  |
| Google AI  | langchain-google-genai   | gemini-2.0-flash           |
| LM Studio  | langchain-openai         | auto (local)               |

Security: keys stay on the server

In this demo, provider API keys are stored in the BFF session only (req.session.langchain_config). They are never included in API responses sent to the browser.

LCEL pattern

LCEL — the new way to chain

Before 0.2.x, LangChain used AgentExecutor + create_openai_functions_agent. These are deprecated. This demo pins LangChain 0.3.x, which hand-builds an LCEL runnable agent as shown below. Note this is what the demo pins, not the current recommendation: as of LangChain 1.x (GA October 2025), the default recommended pattern is create_agent or LangGraph rather than a hand-rolled LCEL agent loop.

# LangChain 0.3.x LCEL pattern (this demo's pinned version)
from langchain_groq import ChatGroq
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder

llm = ChatGroq(model="llama-3.1-8b-instant", api_key=groq_key)

# Bind tools so the model can call them
chain = prompt | llm.bind_tools(tools)

# Tool-calling loop

Why LCEL?

  • Streaming out of the box — every step yields deltas
  • Works with any model that supports tool calling
  • Composable — add output parsers, retry logic, fallbacks as pipe steps
  • Testable — mock any pipe stage independently
